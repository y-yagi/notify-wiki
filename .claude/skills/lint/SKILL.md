`wiki/` 全体を横断的にチェックし、問題点を報告する。CLAUDE.mdの「Health check」の項に対応する。

## チェック項目

1. **矛盾** — 複数のページ間で食い違う記述がないか（例: watch数の上限が `concepts/inotify.md` と `comparisons/*.md` で違う値になっている等）。
2. **古い記述** — `wiki/sources/*.md` に、より新しい情報を持つ資料が後から追加されているのに、対応する `concepts/*.md` の frontmatter の `updated` が更新されていないもの。
3. **孤立ページ** — `wiki/concepts/` や `wiki/comparisons/` 配下で、他のどのページからも `[[name]]` でリンクされていないページ（`wiki/index.md` からのリンクは除く）。
4. **リンク切れ** — 本文中の `[[name]]` が指す先のページが `wiki/concepts/` にも `wiki/comparisons/` にも存在しない。
5. **未取り込みの資料** — `raw/` にファイルはあるが、対応する `wiki/sources/*.md` が存在しない。
6. **未着手の概念** — ページ本文中で言及されているが、まだ独立したページが無い概念（例: "Related concepts" にリンクだけあってページが無い、本文中で名前だけ出てくる等）。
7. **frontmatter不備** — `title` / `tags` / `updated` / `sources` のいずれかが欠けている、または `sources` が指すファイルが存在しない。

## 手順

1. `wiki/` 配下の全ファイルを列挙し、上記の観点でスキャンする。
2. 見つかった問題を、種類ごとにグループ化して一覧で報告する（ファイルパス・該当箇所を明記）。
3. **自動修正はしない**。「直しますか？」と確認するか、次のアクション（`/ingest` で埋める、ユーザーに一次資料を求める等）を提案するに留める。ただしユーザーが明示的に修正を依頼した場合は、その指示された範囲だけ修正してよい。
4. 問題が一つも無ければ「問題なし」と簡潔に報告する。
